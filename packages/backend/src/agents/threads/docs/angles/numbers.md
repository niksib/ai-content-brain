# Numbers

## What this angle is

A post built around one specific number — a count, a dollar amount, a duration, a percentage — that leads to an observation the reader didn't expect. The number is doing the work. Strip the number and the post collapses.

The reader stops because the number is concrete and surprising on its own. The reaction is "wait, really?", "that can't be right", or "I had no idea". Numbers work because most posts make claims without proof; one specific number makes the claim feel verified, even if it's just a single data point.

## Signal in the dump

The dump contains a Numbers angle if the user:
- Mentions a specific count, figure, dollar amount, year, percentage, or duration
- Says something like "took me X days", "spent $Y", "got Z signups", "47% of people"
- Has a real metric from their work, life, or research
- Quotes a number that surprised them

Phrases to look for: any actual digit in the dump, words like "spent", "saved", "took", "made", "lost", "after X years", "in N days".

## Structure

A Numbers post puts the number up front and leads to one observation. The number is the hook. Everything else is short.

**Variant A — Number + observation (1-3 sentences)**

State the number, state what it means. That's it. No setup, no story.

```
[Number stated flat — context tight to it]

[The observation that follows from the number — one line]
```

**Variant B — Two numbers in contrast (2-4 sentences)**

Two numbers placed side by side, with the gap between them carrying the meaning.

```
[Number A — the base case]
[Number B — the comparison]

[The implication — what the contrast means, in one line]
```

**Variant C — Number + short consequence chain (3-5 sentences)**

A number followed by 2-3 short lines that draw out an unexpected consequence.

```
[The number, in context]

[What it means at first glance]

[The twist — the consequence most people miss]
```

## Examples that work

**Number + observation:**

> AI just killed 47,000 jobs in one month. But here's what nobody's talking about: The robots don't pay taxes. And that's going to break everything.

> The Chinese AI agent Manus AI is running 50 social media accounts 24/7 automatically.

> Amazon made their 2,847 engineers spend 8 months documenting every code pattern, every debugging workflow, every optimization trick. Then fed it all to AI. Then fired them.

**Two numbers in contrast:**

> A $2,000 emergency fund means the car repair won't kill you. A $20,000 emergency fund means the job loss won't either.

> Posted daily for 3 months. Reach was flat. Cut to 3x/week with sharper hooks. Reach doubled.

**Number + consequence chain:**

> The average person switches careers at 39. The average new entrepreneur is 42. The average first-time millionaire is 49. You get your creative second wind between 45-55. You are not behind. You are in the sweet spot.

## Banned openings

The number should arrive in the first line, ideally in the first six words. Padding before it kills the angle.

- "I want to share a number that surprised me…"
- "Did you know that…"
- "Here's a wild stat:"
- "Recent research shows that…"
- "According to a study…"
- "Fun fact:"

State the number directly. Citations and framing soften the impact.

## Banned closings

A Numbers post ends on the observation. Adding more dilutes the punch.

- "Crazy, right?"
- "Let that sink in."
- "What do you think this means?"
- A second observation that competes with the first
- A summary that restates the number

The number plus one observation is the whole post. Stop there.

## Anti-patterns

- **Vague number.** "Tons of people" / "a lot of money" / "many years" — these aren't numbers. Use the actual digit or pick a different angle.
- **Made-up number.** Inventing or rounding a number you don't actually know breaks the post the moment a reader checks. If the user didn't provide it, don't insert one.
- **Number without an observation.** "I posted 47 times last month." So what? The number must lead somewhere unexpected.
- **Banal observation.** Number + "wow, that's a lot" doesn't earn a post. The observation should reframe what the number means.
- **Stacking numbers.** Five different stats in one post turns it into a report. One number does more work than five.
- **Number that doesn't survive scrutiny.** If the reader can ask "based on what?" and there's no good answer, the post falls apart. Either the number is real or it isn't.

## Self-check before saving

Read the draft and ask: **if I removed the number, does the post still have a point?** If yes — the number wasn't doing real work and this isn't a Numbers angle. If no — the number is load-bearing, which is exactly right.
