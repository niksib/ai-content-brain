# LinkedIn Post Structures

## What LinkedIn Rewards in 2026

- **Saves**: 5x more reach than a like — structure content people will want to return to
- **Comments**: 2x more reach than a like — end with a question that invites real response
- **Dwell time**: algorithm measures how long people stop — write to be read fully
- **First 60-90 minutes**: early engagement determines 70% of total reach

The first ~200 characters (before "see more") determine everything. If the hook doesn't earn the click, the rest doesn't matter.

## Structure 1: Personal Story with Lesson
**Best for:** Experiences, failures, decisions, turning points
**Length:** 800-1300 characters

```
LINE 1-2 (HOOK — narrative):
Start mid-scene. Something happened.
"I almost [dramatic thing] [yesterday/last week/this morning]."
"Three years ago, I [specific thing]. It was the worst decision I ever made."
Never start with "I" as the first word — weak opener
Strong opener: "[Something happened.] Here's what I learned."

[blank line]

SETUP (2-3 lines):
Minimum context — what was at stake, who was involved
Don't over-explain — trust the reader

[blank line]

THE STORY (3-5 short paragraphs):
One beat per paragraph, max 2-3 lines each
White space between every paragraph — mobile reading
Build toward the turning point or realization
Don't skip the uncomfortable parts — they're the value

[blank line]

THE LESSON (2-3 lines):
Specific, not generic
"What I do differently now: [specific thing]"
Not "always be authentic" — but "I now send the draft to one person first"

[blank line]

CTA (1 line):
One genuine question
"What would you have done?"
"Has this happened to you?"
"Am I missing something here?"
```

## Structure 2: Insight / Contrarian Take
**Best for:** Industry observations, hot takes, opinions that challenge common thinking
**Length:** 600-1000 characters

```
LINE 1-2 (HOOK — contrarian or data):
"Everyone in [field] is obsessed with [X]. Here's why [Y] actually matters more."
"We analyzed [number] [things]. The data shows something most people get wrong."
"[Common belief] is [wrong/incomplete/misunderstood]."

[blank line]

THE SETUP (2-3 lines):
Why people believe the common thing — validate before challenging
"I get it. I used to think this too."

[blank line]

THE ARGUMENT (3-4 paragraphs, 2-3 lines each):
One point per paragraph
Use specific evidence: your experience, data, examples
The argument should be specific enough that someone could disagree with specifics

[blank line]

THE REFRAME (2-3 lines):
What people should think/do instead
Specific and actionable

[blank line]

CTA (1 line):
"Do you see this differently?" — invites genuine pushback
"What's your experience been?" — invites comparison
```

## Structure 3: Framework / Process Post
**Best for:** Systems, methods, processes others can replicate
**Length:** 1000-1300 characters

```
LINE 1-2 (HOOK — outcome-first):
"Here's the [number]-step [process] that [specific result]."
"I've [done X] for [time]. Here's the exact system."
Lead with the outcome before the steps

[blank line]

CONTEXT (1-2 lines):
Who this is for, when to use it
Optional — skip if hook is self-explanatory

[blank line]

THE FRAMEWORK:
Each step on its own line or short paragraph
Bold the step name if using bold
Include the "why" for non-obvious steps
Number them explicitly: "Step 1:", "Step 2:"

[blank line]

THE EDGE CASE (1-2 lines):
"The one thing most people skip..."
"Where this breaks down..."
This rewards people who read to the end

[blank line]

CTA:
Save CTA: "Save this — I use it every [time period]"
or Comment CTA: "What would you add to this?"
```

## Structure 4: Achievement / Milestone
**Best for:** Results, wins, hitting goals — without sounding like a brag
**Length:** 600-1000 characters

```
LINE 1-2 (HOOK — lead with the result, then humanize):
"We hit [milestone] today. I cried a little." 
"[Specific result] happened this week. Here's what nobody tells you about this moment."
The human reaction is what makes achievements relatable

[blank line]

THE CONTEXT (2-3 lines):
Where this started — the before state
Be specific about the difficulty: "We almost quit in [month]"

[blank line]

THE REAL STORY (3-4 paragraphs):
What actually made the difference — not the obvious answers
The decisions, the pivots, the things that almost didn't happen
One specific detail people don't expect

[blank line]

WHAT I'D TELL MYSELF AT THE START (2-3 lines):
The insight in hindsight
Must be specific — not "believe in yourself" but "hire someone for X earlier"

[blank line]

CTA:
"What's one thing you wish you'd known earlier?" — keeps conversation going
```

## Structure 5: Listicle / Resource Post
**Best for:** Tips, tools, resources, observations
**Length:** 800-1200 characters

```
LINE 1-2 (HOOK):
"[Number] [things] that [specific benefit] — and most people overlook [one of them]"
Tease the best item to create curiosity

[blank line]

THE LIST:
Each item on its own paragraph
Format: "1. [Item name] — [specific explanation]"
Include the non-obvious item early (not at the end) — rewards skimmers

[blank line]

THE DEEPER POINT (2-3 lines):
What ties these together
The underlying principle behind the list

[blank line]

CTA:
Save CTA if it's a reference people will return to
Comment CTA: "What would you add?" — grows the resource collaboratively
```

## Formatting Rules

### Always
- One blank line between every paragraph — mandatory for mobile reading
- Short paragraphs: max 2-3 lines
- Contractions: it's, I've, they're, didn't — never formal equivalents
- First word should NOT be "I" — weak opener
- End with a single genuine question

### Never
- External links in the post body (50-60% reach reduction) — link in first comment if needed
- More than 5 hashtags (68% reach reduction)
- Bullet points as the entire post structure — use them sparingly within prose
- AI-sounding words (see anti-ai-writing skill)
- Paragraph that summarizes what was just said
- "In conclusion..." / "To summarize..." endings

### Hashtags
- 3-5 maximum, placed at the end
- Specific over generic: #AIagents not #AI
- Never: #LinkedIn #Post #Content (too generic to add value)
