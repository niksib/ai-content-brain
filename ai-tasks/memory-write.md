# Epic: Cross-session memory write by agents

## Концепт

Сейчас memory blocks юзера заполняются только во время онбординга. Задача — дать агентам в чат-сессиях возможность **записывать новые факты** в память юзера, когда юзер в разговоре упоминает что-то стоящее запомнить для будущих сессий.

## Примеры триггеров

- "Я закончил курс по Python, теперь хочу писать больше про AI/ML"
- "Переключаюсь с консалтинга на продуктовую разработку"
- "Моя аудитория на самом деле не джуны, а мидлы"
- "Убери из pillars тему про фриланс, я перестал этим заниматься"

## Flow

1. Агент замечает факт, который стоит запомнить
2. Вызывает tool `remember` (или `update_memory`) с предложением
3. В чате появляется **inline confirm**: "Запомнить это для будущих сессий?"
   - Показан ключ, заголовок, новое содержимое, diff (если обновление существующего блока)
4. Юзер: **Да / Нет / Поправить формулировку**
5. При подтверждении — запись в `MemoryBlock`

## Обработка конфликтов

Если блок с таким `key` уже существует:
- Агент видит существующее содержимое (через memory-map read tool)
- Предлагает один из вариантов: **replace** / **append as nuance** / **merge**
- Юзер выбирает

## Зависимости

- **Требует**: завершённый epic "Agents memory update" (memory-map + read tool)
- **Не в scope**: автозаписи без подтверждения, история изменений memory blocks

## Открытые вопросы

- UI для inline confirm в чате — отдельный компонент `MemoryWriteConfirm.vue`?
- Отменить записанное через N секунд (undo toast) или только через Creator Profile?
- Rate limit на количество предложений от агента в одной сессии?
